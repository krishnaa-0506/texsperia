💾 DATA STORAGE SYSTEM:

Traditional (WHAT WE REMOVED):
  Frontend → Backend API → MongoDB Cloud

Current (WHAT WE HAVE NOW):
  Frontend ← → Browser localStorage (in-memory)

How it works:
1. User fills registration form
2. Data saved to browser's localStorage
3. Admin logs in locally
4. Admin views data from localStorage
5. Admin exports to CSV

Benefits of this approach:
  ✓ No server needed
  ✓ No port 3001 running
  ✓ No MongoDB connection issues
  ✓ All in one frontend file
  ✓ Perfect for simple applications

Limitations:
  ⚠ Data only in this browser (not shared across browsers)
  ⚠ Data lost if browser cache cleared
  ⚠ No cloud backup (can export to CSV)
